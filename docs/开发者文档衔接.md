### ✅ **1. Card组件heading角色断言问题（已修复）**

**问题根因：**
- CardTitle实际渲染为`<div>`元素，而非`<h1>-<h6>`heading元素
- CardDescription也是`<div>`，不是`<p>`
- 不支持`as="h1"`属性

**修复方案：**
```typescript
// ❌ 修复前
const title = screen.getByRole('heading', { name: /房间管理概览/i })
expect(title.tagName).toBe('H3')

// ✅ 修复后
const title = screen.getByText(/房间管理概览/i)
expect(title).toHaveClass('text-2xl', 'font-semibold')
```

### ✅ **4. AuthGuard的mock函数调用方式（已修复）**

**问题根因：**
- `vi.mocked(useAuthStore)`使用错误
- mock返回对象而非函数，但组件调用`useAuthStore()`作为函数

**修复方案：**
```typescript
// ❌ 修复前
vi.mock('@/lib/stores/useAuthStore', () => ({
  useAuthStore: {
    getState: () => ({...})  // 返回对象
  },
}))
vi.mocked(useAuthStore).mockReturnValue({...})

// ✅ 修复后
const mockLoadFromStorage = vi.fn()
vi.mock('@/lib/stores/useAuthStore', () => ({
  useAuthStore: () => ({  // 返回函数
    isAuthenticated: true,
    loadFromStorage: mockLoadFromStorage,
  }),
}))
```
